# Support

This document explains where to get help and how to route different kinds of requests across the agents-repo organization.

## Questions and usage help

Before opening a new request:

1. Search existing issues and documentation in the relevant repository.
2. Check that repository's README for setup and usage guidance.

For organization-wide policy questions, open an issue on [agents-repo/.github](https://github.com/agents-repo/.github/issues).

## Bug reports

Open an issue in the repository where the bug occurs, using its issue template when available:

- [registry issues](https://github.com/agents-repo/registry/issues/new/choose)
- [webapp issues](https://github.com/agents-repo/webapp/issues/new/choose)
- [registry-proxy issues](https://github.com/agents-repo/registry-proxy/issues/new/choose)

Include reproduction steps, expected behavior, and actual behavior.

## Feature requests

Open an issue in the repository that would implement the feature. Use the feature proposal template when the repository provides one.

## Security issues

Security vulnerabilities are not handled through support channels. Report them privately using [SECURITY.md](SECURITY.md).

## Response expectations

Community support is provided on a best-effort basis. There is no guaranteed response time for general questions or feature requests.
